# RAG

# 📄 AI PDF Chatbot (RAG Powered)

A Streamlit-based conversational AI app that lets you upload a PDF and ask questions about its content using **Retrieval-Augmented Generation (RAG)**.

---

## 🚀 Features

- Upload any PDF file and process it instantly
- Semantic search over document chunks using **FAISS** vector store
- Embeddings powered by `sentence-transformers/all-MiniLM-L6-v2` via HuggingFace
- LLM responses generated by **LLaMA 3.1 8B Instant** via Groq
- Clean, interactive chat interface built with Streamlit

---

## 🧱 Tech Stack

| Layer | Tool |
|---|---|
| UI | Streamlit |
| PDF Parsing | PDFPlumber (via LangChain) |
| Text Splitting | RecursiveCharacterTextSplitter |
| Embeddings | HuggingFace `all-MiniLM-L6-v2` |
| Vector Store | FAISS |
| LLM | Groq — `llama-3.1-8b-instant` |
| Orchestration | LangChain |

## 🔧 Setup

### 1. Clone the repository

```bash
git clone https://github.com/your-username/ai-pdf-chatbot.git
cd ai-pdf-chatbot
```

### 2. Install dependencies

```bash
pip install -r requirements.txt
```

### 3. Configure secrets

Create a `.streamlit/secrets.toml` file in the project root:

```toml
HUGGINGFACEHUB_API_TOKEN = "your_huggingface_token"
GROQ_API_KEY = "your_groq_api_key"
```

> Get your Groq API key at [console.groq.com](https://console.groq.com)  
> Get your HuggingFace token at [huggingface.co/settings/tokens](https://huggingface.co/settings/tokens)

### 4. Run the app

```bash
streamlit run app.py
```

## 🖥️ Usage

1. Open the app in your browser (default: `http://localhost:8501`)
2. Upload a PDF file using the file uploader
3. Click **"Start Conversation"** to process and index the document
4. Type your question in the text input and click **"Predict"**
5. The app retrieves the most relevant chunks and generates an answer

## ⚙️ How It Works

```
PDF Upload
    │
    ▼
PDFPlumberLoader → raw text per page
    │
    ▼
RecursiveCharacterTextSplitter (chunk_size=100, overlap=20)
    │
    ▼
HuggingFace Embeddings → vector representations
    │
    ▼
FAISS Vector Store (stored in session state)
    │
    ▼
User Query → similarity_search (top-5 chunks)
    │
    ▼
PromptTemplate → ChatGroq (LLaMA 3.1 8B) → Answer
```

---

## 📝 Notes

- Chunk size is set to `100` characters with `20` overlap — suitable for dense technical PDFs. Increase `chunk_size` (e.g. `500–1000`) for better context in general documents.
- The vector store is stored in `st.session_state`, so re-uploading a new PDF resets it.
- The app uses `tempfile` to safely handle uploaded PDFs without writing to disk permanently.

## 🛡️ Security

- API keys are stored in Streamlit secrets and never exposed in the source code.
- Add `.streamlit/secrets.toml` to your `.gitignore` before pushing to a public repository.

```gitignore
.streamlit/secrets.toml
```
